What Gets Built — Detail
1. RSC Streaming Layout

page.tsx is a Server Component rendering DashboardGrid with independent <Suspense> boundaries per widget — each widget is its own async Server Component fetching its own slice of data in parallel (no waterfall). This is the textbook Next.js 14 streaming pattern: page shell paints instantly, each widget pops in as its query resolves.
loading.tsx is the full-grid skeleton (all widgets in skeleton form) shown only on hard navigation; the per-widget Suspense fallbacks handle in-place streaming.

2. Widget Grid (dense, Linear-style)

12-column CSS grid: MyCommitmentsWidget (span 6), UpcomingMeetingsWidget (span 6) on row 1; TeamPulseWidget (span 4) + RecentActivityFeed (span 8) on row 2.
Every widget is a Card with a thin border (no shadow, no gradient header) — header row has a small label + a "View all →" link (text-only, no button chrome).

3. MyCommitmentsWidget

Server-fetched top 5 (PENDING first, then MISSED), each row: status StatusDot + truncated text + RelativeTime due date + owner Avatar (self, so just shown for consistency with team views later).
Entire row is a <Link> to /commitments/:id — no separate "view" button, the whole card row is the click target (cursor-pointer + hover bg), matching the "every item clickable" principle.

4. UpcomingMeetingsWidget

Server-fetched next 3 scheduled meetings: platform icon + title + relative scheduled time + a Badge for status (SCHEDULED/BOT_JOINING/RECORDING).
Click → /meetings/:id.

5. TeamPulseWidget

Single big number (fulfillment rate %) + a tiny inline sparkline (lightweight SVG, no chart library yet — Recharts arrives Day 34) + trend arrow (↑ improving / ↓ declining text, color via text-foreground + icon, not loud red/green blocks).

6. RecentActivityFeed

Client Component (will later subscribe to Socket.io in Day 39) wrapped in its own Suspense; initial data from server, rendered as a dense vertical list inside a ScrollArea (max-height capped so the dashboard doesn't grow unbounded) — each ActivityFeedItem is one line: icon, actor, action text, RelativeTime.

7. QuickActionsRow

Two small pill buttons under the page header: "Add meeting" (stub click → toast "Coming Day 32"), "Invite teammate" (stub). Establishes the pattern that every page header gets action buttons, even before the modals exist.
