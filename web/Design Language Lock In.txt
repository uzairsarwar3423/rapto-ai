Design Language Lock-In (applies to all 5 days)
COLORS (neutral, no gradients):
  --background:       0 0% 100%        (light) / 0 0% 7%   (dark)
  --surface:          0 0% 98%         / 0 0% 10%
  --border:           0 0% 90%         / 0 0% 16%
  --foreground:       0 0% 9%          / 0 0% 95%
  --muted-foreground: 0 0% 45%         / 0 0% 60%
  --accent:           shadcn neutral or single brand hue at low saturation
  --hover:             background + 4% black/white overlay (no color shift)

TYPOGRAPHY:
  Headings: Inter/Geist, 13–14px for UI labels (NOT 16px+ marketing sizes)
  Body/table text: 13px, line-height 20px
  Monospace: IDs, keyboard shortcuts (JetBrains Mono / Geist Mono)

SPACING:
  Sidebar width: 240px expanded / 56px collapsed (icons only)
  Row height (tables/lists): 36px dense, 44px comfortable (toggle)
  Topbar height: 48px (NOT 64px — keep dense)

MOTION:
  Only: opacity, transform (translateY 4px), 120–160ms ease-out
  NEVER: scale bounce, spring physics, color-shifting hovers
  Sidebar collapse: 180ms width transition
  Command menu open: 100ms fade + 4px slide

INTERACTION RULES:
  - Every row: hover bg + cursor-pointer, click anywhere = open
  - Every list: arrow keys navigate, Enter opens, Esc closes
  - Cmd+K always available, always fast (<50ms to open)
  - No modal for anything that can be a Sheet/Drawer (preserve context)

What Gets Built — Detail
1. Sidebar (collapsible, hover-aware)

Default width 240px. Collapse button shrinks to 56px (icon-only), persisted in a cookie (read server-side in layout.tsx to avoid layout flash — same pattern as shadcn's official sidebar block).
Collapsed state: icons only, full label appears in a Tooltip on hover (delay 300ms, no animation bounce).
Sections: Workspace group (Dashboard, Meetings, Commitments, Action Items, Team, Analytics) and a divider then Intelligence, then bottom-pinned Settings + SidebarUser.
Active route highlighted with a subtle left-border accent + bg tint — no bold color blocks.
SidebarTeamSwitcher at top: avatar + team name + chevron → DropdownMenu with team plan badge, "Create team" (future), "Team settings" link.
Keyboard: Cmd+\ toggles collapse (registered via useKeyboardShortcut).

2. Topbar (48px, dense)

Left: Breadcrumb (auto-derived from route segments via a small path-to-label map).
Center-left: SearchTrigger — a pill that says "Search everything…" with a Kbd showing ⌘K. Clicking or pressing the shortcut opens CommandMenu.
Right: NotificationBell (opens a Sheet sliding from the right — wired with placeholder content today, real data Day 39), then TopbarActions slot (empty today, pages inject buttons later, e.g. "New Meeting").
No page title duplication in Topbar — title lives in PageHeader inside the content area (Linear pattern: topbar = navigation/global, page header = content-specific).

3. Command Menu (Cmd+K) — the soul of "keyboard-first"

Built on cmdk (shadcn's command.tsx).
Global listener mounted once in AppShell, toggled via useKeyboardShortcut('cmd+k').
Groups today: Navigate (Dashboard, Meetings, Commitments, Action Items, Team, Analytics, Settings), Actions (placeholder "New Meeting" — wired Day 32).
Fuzzy filter built-in via cmdk. Esc closes. Arrow keys + Enter navigate.
Registry pattern (commandMenu.registry.ts) so future days (Day 28+) just push() new commands instead of touching the component.

4. AppShell (RSC)

app/(dashboard)/layout.tsx is a Server Component: fetches user + team via existing server API client (cookie-forwarded), redirects to /login or /onboarding if missing (reusing Day 10 guards).
Passes user/team as props into AppShell, which renders <SidebarProvider> (cookie-read default state) → Sidebar + a flex column of Topbar + <main>{children}</main>.
<main> gets a ScrollArea so independent page scroll never moves the sidebar/topbar.

5. PageContainer / PageHeader

PageContainer: max-w-[1400px] mx-auto px-6 — used by literally every dashboard page going forward.
PageHeader: <h1> (15px semibold) + optional subtitle (13px muted) + actions slot (right-aligned button row) — every future page (meetings, commitments, etc.) wraps its top in this.
